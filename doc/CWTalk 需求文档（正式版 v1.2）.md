## 2. 硬件接口架构

### 2.1 CAT 接口（只读）

| 项目         | 说明                                                         |
| ------------ | ------------------------------------------------------------ |
| **用途**     | 读取实时频率（MHz）用于界面显示与 ADIF 记录                    |
| **实现方式** | `ICatReader` 抽象 + `CatReaderFactory`；**已实现 Yaesu**（FT-710 等，`FA`/`FB` 读频） |
| **物理接口** | 串口（COM）或 USB 虚拟串口（Yaesu 常用 **Enhanced COM**）    |
| **数据获取** | 频率（MHz，界面三位小数）；模式固定为 CW 写入日志              |
| **协议范围** | **已实现**：Yaesu（`Cat/Backend=yaesu`）。**规划**：Icom CI-V（`Cat/Backend` 扩展） |
| **开关**     | `Cat/Enabled`：关闭则不创建读频器、频率框可手动输入            |
| **注意**     | **不用于 PTT 或键控**；与键控可**同口共享**（见下）          |

**技术细节（已实现）：**
- Qt `QSerialPort`；默认波特率 **38400**（`Hardware/CAT_Baud`，与 FT-710 CAT-1 菜单一致）
- 轮询间隔 `Cat/Poll_Interval_Ms`（默认 800ms），单次超时 `Cat/Timeout_Ms`（默认 1000ms）
- **键控发射期间暂停 CAT 轮询**（`PCKeyer::transmissionActiveChanged`），避免半双工口冲突；发射结束恢复
- **CAT 与 Keying 同口**时：每次 CAT 事务前 `releaseKeyingLines()` 释放 DTR/RTS
- 读频成功时频率框只读并高亮；失败或未启用时可手动输入 MHz
- **频率变化**（CAT 或手动确认后变化）：**自动清除当前 QSO 字段**（频率保留在频率框）

### 2.2 Keying 接口（写）

| 项目         | 说明                                      |
| ------------ | ----------------------------------------- |
| **方案**     | 串口 DTR/RTS 线控                         |
| **电平极性** | **可配置**：高电平有效 / 低电平有效       |
| **选择线**   | DTR 或 RTS 可选                           |
| **物理接口** | 物理串口或 USB-串口转接（需外部隔离电路） |
| **侧音**     | **不通过电脑生成**，完全依赖电台侧音      |

**技术细节：**
- 使用 Qt `QSerialPort` 控制 DTR/RTS 状态
- 软件生成精确时序（点/划比例 1:3，符号间隔 1 点，字母间隔 3 点，单词间隔 7 点）
- 实时速度调节范围：5-40 WPM
- 打断响应延迟 < 50ms

- [ ] WinKeyer 支持（K1EL 芯片）

## 3. 核心功能模块

### 3.1 发码控制（Keying）

| 功能         | 说明                                                         |
| ------------ | ------------------------------------------------------------ |
| **输入方式** | 键盘直接输入、快捷键按钮（F1-F8），支持键盘 F1-F8 直接触发       |
| **速度控制** | WPM 实时可调（5-40 WPM）                                     |
| **打断机制** | Esc 键立即停止                                               |
| **宏变量**   | `<call>`, `<his_rst>`, `<name>`, `<qth>`                     |

**宏变量说明：**
- `<call>`：对方呼号（呼号输入框实时内容）
- `<his_rst>`：给对方的信号报告（发送 RST 输入框内容）
- `<name>`：己方姓名（从配置文件 `Station/Name` 读取）
- `<qth>`：己方地址（从配置文件 `Station/QTH` 读取）

**快捷键按钮交互：**
- **单击按钮** 或 **按键盘 F1-F8**：将内容追加到发送输入框末尾，自动启动发送
- **双击按钮**：启动该快捷键内容的自动循环发送（发送完成后等待间隔再次发送）
- **右键按钮**：弹出对话框编辑该快捷键的标题和内容

### 3.2 自动循环发送

| 项目         | 说明                                  |
| ------------ | ------------------------------------- |
| **触发**     | 双击 F1~F8 快捷键按钮                 |
| **报文**     | 对应快捷键的自定义内容                |
| **循环间隔** | 2-10 秒可调（默认 4 秒），配置项 `Operation/CQ_Interval` |
| **打断方式** | ① 双击正在循环的按钮 ② 按 Esc 键 ③ 在 Call 输入框输入内容 ④ 点击其他宏按钮 |
| **状态显示** | 循环中的按钮变为绿色背景高亮          |

**循环行为：**
- 每次循环发送前**清空输入框**，避免内容累积
- 发送完成后进入等待间隔
- 间隔结束后自动再次发送（重新展开宏变量，如 `<call>` 等会取最新值）

### 3.3 通联记录（Logging）

**存储格式：** ADIF v3 文本（`.adi` / `.adif`），路径 `Files/ADIF_Path`（默认程序目录 `log.adif`）

**生命周期：**
- 启动：`QsoLog::load` 全量读入内存索引
- 记录：**仅追加单条**到文件末尾并 `flush`（非整文件重写）
- 重复判定：同 `qso_date`+`time_on`+`call` 等关键字段已存在则拒绝追加

**记录字段（已实现）：**

| ADIF 字段  | 来源                         | 说明 |
| ---------- | ---------------------------- | ---- |
| CALL       | 呼号框（自动大写）           | 必填 |
| QSO_DATE   | `time_on` 对应 UTC 日期      | YYYYMMDD |
| TIME_ON    | **首次在呼号框输入**时记下     | HHMMSS（6 位，无分隔符） |
| TIME_OFF   | 点击「记录」时               | HHMMSS |
| FREQ       | 频率框 / CAT                 | **MHz**，三位小数（符合 ADIF 3.1.4 Megahertz）；旧日志若误存为 ≥1000 的整数 kHz，读入时自动 ÷1000 |
| MODE       | 固定                         | `CW` |
| RST_SENT   | 发送 RST；空则记 `599`       | 最多 3 位数字 |
| RST_RCVD   | 接收 RST；空则记 `599`       | 同上 |
| NAME/QTH/COMMENT | 有内容才写入           | 可选 |
| MY_RIG     | `Station/MY_RIG`             | 有内容才写入；序列化时 **`my_rig` 放在记录末尾** |
| OPERATOR   | —                            | **不写入** |
| BAND       | —                            | v1.2 **未实现**（规划由频率推导） |
| MY_POWER   | —                            | v1.2 **未实现** |

**操作（已实现）：**
- 主界面 **「记录」按钮** 或 **Ctrl+L**
- 校验呼号非空、`Record::iscomplete()`（至少含 date/time_on/freq/mode/call/rst_sent/rst_rcvd）
- 成功后：**清空通联区**（呼号、RST、姓名、QTH、备注、发送框）；**频率保留**；状态栏提示条数与路径

### 3.4 历史通联查询

| 项目         | 说明 |
| ------------ | ---- |
| **触发**     | 呼号框 **失焦**（`editingFinished`）且呼号满足「完整」规则（≥3 字符、字母数字与 `/`、含字母） |
| **数据源**   | 已加载 ADIF，`find_records_by_call`（**不区分大小写**），最多 **5 条** |
| **排序**     | **时间逆序**（最新在上） |
| **展示**     | 主窗口底部 **`qsoHistoryPanel`**（非表格）：标题为呼号；正文最多 5 **文本行**，等宽字体，可选中复制 |
| **行格式**   | 逗号分隔，**无字段名**。固定段：`日期, time_on, freq, mode`；`time_on` 为 **6 位数字**（如 `115300`）。可选段：`name`、`qth`、`comment` — **仅非空时追加**，避免多余逗号 |
| **示例**     | `2024-05-10, 115300, 50.284, CW, Taro, Tokyo` |
| **隐藏**     | 呼号清空/未完整、修改呼号与当前展示不一致、点击「记录」或「清除」时隐藏 |
| **自动填充** | v1.2 **未实现**（规划可选） |

### 3.5 呼号前缀查询（新增）

| 项目         | 说明                                                         |
| ------------ | ------------------------------------------------------------ |
| **触发**     | 呼号输入框输入时实时解析前缀                                 |
| **显示内容** | 国家/地区、CQ 分区、ITU 分区、**天线指向角度**（Azimuth）、**距离**（km） |
| **计算基准** | 用户设置的本地 6 位 Grid（Locator，如 PM01bm）               |
| **数据来源** | **内置 CTY 格式数据库 + 大国家国内分区细化规则**             |
| **网络依赖** | **无，完全离线**                                             |

**大国家国内分区细化规则：**

| 国家         | 前缀                | 细分规则         | 区域示例                                                     |
| ------------ | ------------------- | ---------------- | ------------------------------------------------------------ |
| **美国**     | K, N, W, AA-AL      | 第 2 位数字/字母 | W**1**=新英格兰, W**6**=加州, K**H**=夏威夷, A**L**=阿拉斯加 |
| **俄罗斯**   | R, U**A**-U**I**    | 第 2 位数字/字母 | R**A**=欧洲俄, R**0**=远东, U**A**0=西伯利亚, U**A**9=乌拉尔 |
| **澳大利亚** | VK                  | 第 3 位数字      | VK**2**=新南威尔士, VK**6**=西澳, VK**8**=北领地             |
| **加拿大**   | VE, VO, VY          | 第 2 位数字/字母 | VE**1**=海洋省, VE**6**=阿尔伯塔, VY**0**=北极               |
| **巴西**     | PY                  | 第 3 位数字      | PY**1**=里约, PY**4**=米纳斯吉拉斯, PY**8**=亚马逊           |
| **阿根廷**   | LU                  | 第 3 位数字      | LU**1**=布宜诺斯艾利斯, LU**8**=巴塔哥尼亚                   |
| **中国**     | B, VR**2**, XX**9** | 第 2 位字母/数字 | B**A**-B**X**=大陆各省, B**S**=海南, VR**2**=香港, XX**9**=澳门 |

**角度计算：**
- 使用 **Maidenhead Grid** 计算大圆路径（Great Circle）
- 各区域对应中心 Grid（CTY 数据库提供）
- 本地 Grid 与用户设置的 6 位 Grid 计算
- 显示：方位角（0-360°）、距离（km）

**数据文件：**
- 内置 `cty.dat` 或编译为二进制/JSON（约 500KB-1MB）
- 支持用户手动更新（替换数据文件）

## 4. 界面设计

### 4.1 布局

```
┌──────────────────────────────────────────────────────────────────┐
│ 呼号 发RST 收RST 姓名 QTH 备注 频率MHz  [清除] [记录 Ctrl+L]      │  <- 第一行
├──────────────────────────────────────────────────────────────────┤
│ 📍 前缀信息（国家 / CQ / ITU …）                                  │  <- 第二行
├──────────────────────────────────────────────────────────────────┤
│ [摩尔斯发送输入框 — 输入即发送]                                   │  <- 第三行
├──────────────────────────────────────────────────────────────────┤
│ [F1] … [F8]                                                       │  <- 第四行
├──────────────────────────────────────────────────────────────────┤
│ （可选）JH1ABC                                                    │  <- 历史面板标题 = 呼号
│ 2024-05-10, 115300, 50.284, CW, …                                 │  <- 最多 5 行逗号分隔文本
└──────────────────────────────────────────────────────────────────┘
```

**布局说明（从上到下）：**
1. **第一行**：呼号、发送/接收 RST、姓名、QTH、备注、**频率（MHz）** + **清除（Esc）** + **记录（Ctrl+L）**
2. **第二行**：呼号前缀信息（v1.2 为简易规则；完整 CTY 为规划）
3. **第三行**：摩尔斯发送框（`QTextEdit`），输入即发送
4. **第四行**：快捷键 F1–F8
5. **历史面板**（`qsoHistoryPanel`）：呼号失焦且有历史时显示于宏按钮下方，否则隐藏

### 4.2 视觉规范

| 项目         | 要求                                   |
| ------------ | -------------------------------------- |
| **配色**     | 深色模式为主，高对比度，适合夜间操作   |
| **字体**     | 等宽字体显示摩尔斯码，历史记录清晰易读 |
| **窗口尺寸** | 固定或可调，确保一屏完整显示           |
| **状态指示** | CAT 连接状态、CQ 状态、Keying 状态     |

## 5. 配置与设置

### 5.1 硬件设置

| 配置键 / 项目        | 说明 |
| -------------------- | ---- |
| `Cat/Enabled`        | 启用 CAT 读频 |
| `Cat/Backend`        | 读频后端（`yaesu` / `yaesu_ft710` / `ft710`） |
| `Cat/Poll_Interval_Ms` | 轮询间隔（默认 800） |
| `Cat/Timeout_Ms`     | 单次读频超时（默认 1000） |
| `Hardware/CAT_Port`  | **下拉枚举**系统串口；配置端口未插入时显示「(当前未检测到)」 |
| `Hardware/CAT_Baud`  | 波特率（默认 **38400**） |
| `Hardware/CAT_Address` | CI-V 地址（预留，Yaesu 未用） |
| `Hardware/Keying_Port` | 键控串口（下拉，规则同 CAT） |
| `Hardware/Keying_Line` | DTR / RTS |
| `Hardware/Keying_ActiveLow` | 低电平有效 / 高电平有效 |

打开「选项」并确定后，串口与 CAT/键控相关项**热重载**（`initHardware`）。

### 5.2 自动循环设置

- 循环间隔：2-10 秒（默认 4 秒），配置键 `Operation/CQ_Interval`
- 间隔结束后自动再次发送对应快捷键内容

### 5.3 日志与位置设置

| 配置键 / 项目   | 说明 |
| --------------- | ---- |
| `Files/ADIF_Path` | ADIF 日志路径（选项「文件」页可浏览选择） |
| `Station/MY_CALL` | 己方呼号（宏等） |
| `Station/MY_RIG`  | 写入 ADIF 的 `my_rig` |
| `Station/MY_NAME` / `MY_QTH` | 己方姓名/地址（宏 `<name>` `<qth>` 等） |
| `Station/Grid`    | 6 位 Grid（前缀距离/方位规划用） |
| `Station/Power`   | 功率（配置保留；ADIF `tx_pwr` v1.2 未写） |

### 5.4 快捷键宏编辑（F1-F8）

- **数量**：8 个快捷键（F1-F8）
- **每个快捷键包含**：标题（按钮显示文字）+ 内容（发送文本）
- **编辑方式**：右键点击对应按钮，弹出编辑对话框
- **支持变量**：`<call>` `<his_rst>` `<name>` `<qth>`
- **配置存储**：`Shortcuts/F{N}_Title` 和 `Shortcuts/F{N}_Content`（N=1~8）

### 5.5 前缀数据库

- 内置 `cty.dat` 或二进制格式
- 显示数据版本日期
- 支持手动更新（文件替换）
